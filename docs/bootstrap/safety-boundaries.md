# Safety Boundaries

## Allowed changes in the current phase

- create planning and architecture documents
- create the new top-level showcase folder structure
- update repository positioning and navigation docs
- add placeholder project folders for curated extraction work

## Off limits in the current phase

- breaking moves of the runnable app without a migration milestone
- deleting legacy files that still support the current local workflow
- mixing unrelated cleanup into the showcase refactor
- making unverifiable claims about production readiness or organizational standardization

## Dependency policy

Dependency changes are not part of the bootstrap milestone.

## Network policy

Network use is allowed only for explicit reference lookup when needed for planning. Implementation should assume local-first work.

## Required proofs

- visible repository tree reflects the new canonical folders
- planning artifacts exist on disk in the repository
- the existing runnable app layout still remains present after scaffolding
