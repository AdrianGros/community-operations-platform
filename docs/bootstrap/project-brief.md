# Project Brief

## Project name

Community Operations Platform Showcase Hub

## Goal in one sentence

Turn the current repository into a curated showcase repository that presents multiple projects and extracts in a structured, reviewable, portfolio-ready way.

## Measurable end state

- the repository has one clear top-level information architecture
- each showcased project has one canonical folder under `projects/`
- architecture, evidence, and narrative documents live in predictable locations
- runnable artifacts and non-runnable extracts are clearly separated
- the root README explains how a reviewer should navigate the repository in less than two minutes

## Technologies and environment

- GitHub repository curation
- Markdown-first planning and documentation
- existing Python / PySide6 showcase app artifacts
- existing anonymized extract material from the broader community operations platform

## Timeframe

Incremental refactor across milestones, starting with planning and structure scaffolding before content migration.

## Non-goals

- no full rewrite of the existing app during bootstrap
- no uncontrolled mass move of files in a single step
- no claim that the repository already represents the full platform
- no dependency or infrastructure redesign unless a later milestone explicitly approves it
