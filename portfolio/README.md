# Portfolio Layer

This folder is the cross-project positioning layer for the repository.

Its job is to explain how the repository should be read as a showcase hub instead of as a single isolated code sample.

## Intended audience

- hiring managers who need a quick orientation
- engineering reviewers who want curated technical depth
- operations-minded reviewers who care about structure, safeguards, and migration discipline

## How to read this repository

1. start at the root [`README.md`](../README.md)
2. enter the main project at [`projects/community-operations-platform`](../projects/community-operations-platform)
3. choose between overview, extracts, evidence, and roadmap depending on review depth

## Curation rules

- each showcased project gets one canonical folder under `projects/`
- project-specific narrative should not live primarily at root
- evidence should explain selection, relevance, and sanitization
- extracts should maximize technical signal while minimizing publication risk

## Future role of this folder

As additional showcase projects are added, this folder should hold:

- portfolio positioning
- cross-project comparison
- reviewer routing guidance
- project index pages

For the current phase, the canonical planning baseline still lives in [`docs/bootstrap`](../docs/bootstrap).
