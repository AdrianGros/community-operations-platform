# Release Notes

This extract is shaped for review, not deployment.

What it keeps:

- code paths that show wiring, migrations, guards, workers, and monitoring
- enough documentation to understand why those parts matter

What it drops:

- operational runbooks tied to a real host
- environment files and service manager details
- production evidence that contains infrastructure fingerprints

If this package is expanded later, the default rule should stay the same: add only artifacts that carry clear technical value.
